My Automation Project

This project was developed on Toggle plan platform. Toggle plan platform is a user engagement platform that gives you a visual overview of what your team is up to. You can divide tasks between different users and see what the status of each user's tasks is

The Goal:

Create test cases and secnarios, in order to test Toggle plan platform's features and functionality, and build solid automation using Java that tests various use cases and generate reports that display a very concise representation of what have been tested.

Selenium

Selenium is a portable framework for testing web applications. Selenium provides a playback tool for authoring functional tests without the need to learn a test scripting language (Selenium IDE). It also provides a test domain-specific language (Selenese) to write tests in a number of popular programming languages, including C#, Groovy, Java, Perl, PHP, Python, Ruby and Scala. The tests can then run against most modern web browsers. Selenium runs on Windows, Linux, and macOS. It is open-source software released under the Apache License 2.0.

Page object model

A design pattern, popularly used in test automation that creates Object Repository for web UI elements. The advantage of the model is that it reduces code duplication and improves test maintenance. Under this model, for each web page in the application, there should be a corresponding Page Class. This Page class will identify the WebElements of that web page and also contains Page methods which perform operations on those WebElements. Name of these methods should be given as per the task they are performing, i.e., if a loader is waiting for the payment gateway to appear, POM method name can be waitForPaymentScreenDisplay().
